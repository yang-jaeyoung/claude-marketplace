# /cw:design - Design Phase

Create structured design documents for UX/UI or system architecture.

## Usage

```bash
/cw:design --ui      # UX/UI design (Designer agent)
/cw:design --arch    # Architecture (Architect agent)
/cw:design --all     # Both
/cw:design           # Interactive mode
```

## Design Types

| Flag | Agent | Output |
|------|-------|--------|
| `--ui` | Designer | `.caw/design/ux-ui.md` |
| `--arch` | Architect | `.caw/design/architecture.md` |
| `--all` | Both | Both files |

## Workflow

1. **Context**: Check `.caw/brainstorm.md`, existing patterns
2. **Invoke Agent**: Designer or Architect via Task tool
3. **Generate**: Write design documents

## UX/UI Design (`--ui`)

Creates:
- User flow diagrams
- ASCII wireframes
- Component specifications
- Interaction states
- Accessibility requirements

## Architecture Design (`--arch`)

Creates:
- System component diagrams
- Data models (ERD)
- API specifications
- Technical decision records
- Security considerations
