# Rails 8 + Hotwire Plugin Context

**Plugin:** rails8-hotwire
**Purpose:** Full-stack Rails 8 + Hotwire development with specialized agents, hooks, and comprehensive knowledge base.
**Version:** 1.0.0

## Tech Stack

- Ruby on Rails 8
- Hotwire (Turbo + Stimulus)
- Solid Trifecta (Queue, Cache, Cable)
- Kamal 2 (Deployment)
- RSpec (Testing)
- Devise + Pundit (Auth)

## Available Agents

| Agent | Model | Purpose |
|-------|-------|---------|
| `rails-architect` | Opus | Architecture design, complex debugging |
| `rails-architect-low` | Haiku | Quick Rails questions |
| `rails-executor` | Sonnet | Feature implementation |
| `rails-executor-high` | Opus | Complex migrations, refactoring |
| `hotwire-specialist` | Sonnet | Turbo/Stimulus patterns |
| `hotwire-specialist-high` | Opus | Complex real-time systems |
| `rspec-tester` | Sonnet | RSpec test writing |
| `rspec-tester-low` | Haiku | Simple test generation |
| `kamal-deployer` | Sonnet | Kamal/Docker deployment |
| `devise-specialist` | Sonnet | Authentication patterns |
| `activerecord-optimizer` | Sonnet | Query optimization, N+1 fixes |
| `stimulus-designer` | Sonnet | Stimulus controller design |
| `turbo-debugger` | Opus | Turbo response debugging |
| `rails-migrator` | Sonnet | Database migrations |
| `rails-reviewer` | Opus | In-depth code review |
| `rails-reviewer-low` | Haiku | Quick convention checks |

## Automated Hooks

| Hook | Event | Function |
|------|-------|----------|
| rails-detector | SessionStart | Detect Rails project, inject context |
| convention-validator | UserPromptSubmit | Detect Rails anti-patterns |
| migration-guard | PreToolUse | Block dangerous migrations |
| test-enforcer | PreToolUse | Enforce test writing |
| turbo-response-check | PostToolUse | Validate Turbo responses |
| deploy-safety | PreToolUse | Check deployment safety |

## Knowledge Base

- **auth/**: Devise, OAuth, 2FA, Pundit authorization
- **background/**: Solid Queue, Sidekiq, job patterns
- **deploy/**: Kamal, Docker, cloud platforms
- **hotwire/**: Turbo Drive/Frame/Stream, Stimulus patterns

## Core Philosophy

| Principle | Description |
|-----------|-------------|
| Convention over Configuration | Follow Rails conventions |
| HTML over the Wire | Server-rendered HTML instead of JSON APIs |
| Server-side First | Progressive enhancement approach |
| No PaaS Required | Solid Trifecta + Kamal self-hosting |
