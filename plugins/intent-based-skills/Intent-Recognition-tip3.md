## AI에게 의도를 더 잘 전달하기 위한 실용 팁

### 1. 핵심 원칙 요약

| 원칙 | 설명 | 예시 |
|------|------|------|
| **구체성** | 모호한 표현 대신 명확한 조건 제시 | ❌ "좋은 식당" → ✅ "강남역 5분 거리, 2인 이탈리안, 3만원대" |
| **맥락 제공** | 배경/목적/대상 명시 | "신입 개발자에게 설명할 용도로" |
| **제약 조건** | 형식, 길이, 포함/제외 사항 | "표 형식으로, 500자 이내, 코드 예시 포함" |
| **단계적 요청** | 복잡한 작업은 분해 | 1단계: 조사 → 2단계: 비교 → 3단계: 추천 |

### 2. 상황별 프롬프트 패턴

#### 2.1 정보 검색/조사

```
❌ 모호한 요청:
"React에 대해 알려줘"

✅ 의도가 명확한 요청:
"React 18의 새로운 기능 중 Concurrent Features를
- 기존 방식과의 차이점
- 실제 사용 예시 코드
- 마이그레이션 시 주의사항
위 3가지 관점에서 설명해줘. 대상은 React 17 경험이 있는 개발자야."
```

#### 2.2 문서 작성

```
❌ "보고서 작성해줘"

✅ "Q4 매출 분석 보고서를 작성해줘.
- 대상: C-level 임원
- 분량: A4 2페이지
- 필수 포함: 전분기 대비 변화율, 주요 원인 3가지, 액션 아이템
- 톤: 간결하고 데이터 중심
- 형식: 서론-본론-결론 구조, 핵심 수치는 굵게 표시"
```

#### 2.3 코드 관련

```
❌ "로그인 기능 만들어줘"

✅ "ASP.NET Core 8 Web API에서 JWT 기반 로그인 구현해줘.
- 환경: .NET 8, Entity Framework Core, SQL Server
- 요구사항: 
  • Access Token (30분) + Refresh Token (7일)
  • BCrypt 비밀번호 해싱
  • 로그인 실패 5회 시 계정 잠금
- 제외: 소셜 로그인, 2FA (다음 단계에서 추가 예정)
- 출력: Controller, Service, DTO 각각 분리된 코드"
```

### 3. 모호성 방지 체크리스트

프롬프트 작성 시 아래 항목들을 점검하세요:

| 체크 | 질문 | 개선 방법 |
|:----:|------|----------|
| ☐ | **누가** 사용/읽을 것인가? | 대상 명시: "초보자용", "기술팀 내부용" |
| ☐ | **왜** 필요한가? | 목적 명시: "면접 준비용", "프로덕션 배포용" |
| ☐ | **어떤 형식**으로? | 포맷 지정: 표, 목록, 코드, 산문 |
| ☐ | **얼마나** 상세하게? | 깊이/분량: "개요만", "상세 구현" |
| ☐ | **제외할 것**은? | 범위 한정: "프론트엔드 제외", "이론 설명 생략" |
| ☐ | **기준**이 있는가? | 평가 기준: "성능 우선", "유지보수성 중심" |

### 4. AI의 명확화 질문 활용하기

AI가 질문을 하면 **귀찮아하지 말고 상세히 답변**하는 것이 좋습니다:

```
사용자: "우리 프로젝트에 캐싱 추가해줘"

AI: "몇 가지 확인이 필요합니다:
     1. 어떤 데이터를 캐싱할까요? (세션/API 응답/DB 쿼리)
     2. 분산 캐시가 필요한가요? (단일 서버 vs 다중 서버)
     3. 현재 기술 스택은?"

✅ 좋은 응답:
"1. 자주 조회되는 상품 목록 API 응답
 2. 현재는 단일 서버지만 3개월 내 스케일아웃 예정이라 Redis 고려
 3. ASP.NET Core 8 + PostgreSQL"

❌ 나쁜 응답:
"그냥 알아서 해줘"
```

### 5. 반복 대화에서 의도 누적하기

복잡한 작업은 한 번에 완벽하게 전달하기 어렵습니다. **점진적 구체화** 전략을 사용하세요:

```
[1차] 큰 그림 제시
"IAM 서버를 모듈러 모놀리스로 설계하고 싶어"

[2차] 방향 확정 후 상세화  
"인증 모듈부터 시작하자. JWT + Refresh Token 방식으로"

[3차] 구현 세부사항
"토큰 저장은 Redis, 블랙리스트 처리도 포함해줘"

[4차] 예외 케이스
"동시 로그인 제한은 3대까지, 초과 시 가장 오래된 세션 만료"
```

### 6. 효과적인 피드백 제공

AI 응답이 의도와 다를 때 **구체적으로** 피드백하세요:

| 피드백 유형 | ❌ 비효과적 | ✅ 효과적 |
|------------|-----------|----------|
| 방향 수정 | "아니 그게 아니라..." | "보안보다 성능 최적화에 초점 맞춰줘" |
| 부분 수정 | "다시 해줘" | "2번 항목만 DI 패턴으로 변경해줘" |
| 스타일 수정 | "더 잘 써줘" | "기술 용어 줄이고 비유를 사용해서" |
| 추가 요청 | "더 해줘" | "에러 핸들링 부분 추가해줘" |

### 7. 실전 템플릿

자주 사용하는 요청 유형별 템플릿:

#### 기술 조사
```
[주제]에 대해 조사해줘.
- 관점: [비교/장단점/트렌드/실무 적용]
- 깊이: [개요/중급/심화]
- 형식: [표/목록/산문]
- 특히 [특정 관심사] 중심으로
```

#### 코드 리뷰
```
아래 코드를 리뷰해줘.
- 중점: [성능/보안/가독성/아키텍처]
- 환경: [기술스택, 버전]
- 수준: [프로덕션/프로토타입/학습용]
- 출력: 문제점 + 개선 코드 예시
```

#### 문서 작성
```
[문서 유형]을 작성해줘.
- 독자: [대상]
- 목적: [설득/정보전달/가이드]
- 톤: [공식/캐주얼/기술적]
- 길이: [분량]
- 필수 포함: [항목들]
```

### 핵심 요약

> **"AI는 독심술사가 아니다"** — 의도가 명확할수록 결과가 좋아집니다.

1. **5W1H**로 맥락 제공 (Who, What, Why, When, Where, How)
2. **제약 조건** 명시 (형식, 길이, 포함/제외)
3. AI의 **명확화 질문에 성실히 응답**
4. **점진적 구체화**로 복잡한 작업 분해
5. **구체적 피드백**으로 방향 수정
